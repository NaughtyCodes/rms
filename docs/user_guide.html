SaaS Architecture

Multi-Tenant Architecture

Tracly is built as a multi-tenant SaaS platform. This means a single installation can serve multiple independent shops (tenants), each with completely isolated data, configuration, and users.

What is Multi-Tenancy?

Multi-tenancy lets you run many independent shops from one system. Each shop (tenant) has:

Isolated DataProducts, invoices, customers, and settings are completely separate between tenants. Shop A cannot see Shop B's data.
Independent ConfigEach shop has its own name, logo, tax settings, branches, and print styles.
Separate UsersEach tenant has its own set of admins and cashiers. Usernames are unique per tenant.

Login & User Roles

The system supports three distinct user roles, each with specific access levels and capabilities.

Super Admin

Global platform administrator. Manages all tenants and creates new shops.

Tenant Admin

Shop-level administrator. Manages inventory, taxes, and branches for their shop.

Cashier

Branch-level operator. Handles billing and views invoices for their branch.

Step-by-Step Workflows

1. Adding Inventory

Navigate to Inventory in the sidebar.
Click the + Add Product button.
Fill in the product details and click Save Product.

2. Selling Stock

Go to Billing.
Scan or search for items and add them to the cart.
Select Payment Mode and click Create Bill.
Pro Tip: Use the live preview in Print Config to adjust your bills before printing.

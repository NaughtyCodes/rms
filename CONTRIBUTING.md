# Contributing to Tracly

First off, thank you for considering contributing to Tracly! It's people like you that make Tracly such a great tool for the community.

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please create an issue on GitHub. Before creating a new issue, check if it's already reported.

### Suggesting Enhancements
Enhancement suggestions are tracked as GitHub issues. When creating an enhancement suggestion, describe the feature you'd like to see and why it would be useful.

### Pull Requests
1. Fork the repository.
2. Create a new branch for your feature or bug fix.
3. Make your changes.
4. Ensure your code follows the existing style.
5. Submit a pull request.

## Local Development Setup

### Backend (server)
1. Navigate to the `server` directory.
2. Run `npm install`.
3. Start the server with `npm run dev` or `node src/app.js`.

### Frontend (client)
1. Navigate to the `client` directory.
2. Run `npm install`.
3. Start the application with `npm start`.

## Code Standards
- Use ES6+ syntax.
- Follow the existing formatting and naming conventions.
- Write clear, descriptive commit messages.

Thank you for your contribution!
